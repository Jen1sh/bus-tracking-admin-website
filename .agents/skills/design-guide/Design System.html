Design System · v1.0

A calm, trustworthy system for real-time school transport.

Every token here serves one job: answer "Where is the bus right now?" instantly and reduce parent anxiety. Built mobile-first for low-end Android, outdoor readability, and one-handed use.

Engineering

Component library structure

Atomic, reused across Parent / Driver / Admin. State is owned by a single trip store; components are presentational and driven by props.
Primitives → composites
PrimaryButton · StatusBadge · Avatar · Icon
↳ BusCard · NotificationCard
↳ TripTimeline · AlertBanner
↳ MapBottomSheet · LiveMap
Reuse & state strategy
  • LiveMap is shared verbatim by Parent, Driver and Admin — only overlays differ.
  • StatusBadge maps one trip-state enum to a tone, guaranteeing consistent language.
  • A single trip store (phase + progress) feeds ETA, distance, timeline and badges — one source of truth.
  • Components are theme-driven via CSS variables; theming the whole app = swapping tokens.
