# Admin API - Frontend Doc Source

Source for generating admin-facing API docs. Built from .agent/plans/feature/admin-onboarding-rework/ and .agent/plans/feature/admin-console-parity/ (Plan.md + implementation.md), the actual controllers in src/main/java/com/school/bus_tracking/{admin,driver,parent,student,trip}/\*\* + superadmin/, and screen names in bus-tracking/v3/\*.html (Admin Dashboard, Admin Bus/Fleet, Admin Driver(s), Admin Student(s), Admin Parent(s), Admin Trip(s), Super Admin, Super School).

admin-console-parity closed the standalone-resource gaps left by admin-onboarding-rework: Drivers, Parents, Students, and Trips each now have their own list+detail endpoints (§2a, §3, §4). See "Resolved gaps" at the bottom for what changed since the previous version of this doc.

All endpoints: base path as shown, Authorization: Bearer &lt;accessToken&gt;, ADMIN role + tenant-scoped by JWT schoolId unless noted. Response envelope everywhere: { success, data, message } / error: { success: false, error: "ERROR_CODE", message }.

## 2\. Bus management (screens: **Admin Fleet** list, **Admin Bus** detail)

Replaces the old fleet-wide XLSX import. Bus creation and edits now happen directly in-app.

| Feature                                          | Endpoint                                                                                    | Body / notes                                                                                                                                                                                                                                                                                                  |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Create bus (+ driver + empty route, one call)    | POST /api/admin/buses                                                                       | BusCreateRequest: plate\*, displayId?, capacity?, shift?, status?, driverName\*, driverPhone\*, routeName\*, routeCode?. Duplicate plate → 409 PLATE_ALREADY_EXISTS. displayId auto-generated (BUS-&lt;id&gt;) if omitted. Driver name is set once here and **never editable afterward, anywhere** - see §2a. |
| ---                                              | ---                                                                                         | ---                                                                                                                                                                                                                                                                                                           |
| Bus detail (fields + roster)                     | GET /api/admin/buses/{busId}                                                                | → BusDetailResponse (includes students\[\])                                                                                                                                                                                                                                                                   |
| ---                                              | ---                                                                                         | ---                                                                                                                                                                                                                                                                                                           |
| Edit bus (capacity/shift/status/route code+name) | PATCH /api/admin/buses/{busId}                                                              | BusUpdateRequest - partial. **Plate is never editable here. Driver name/phone are no longer part of this call** - edit the driver's phone via §2a instead; BusUpdateRequest dropped both fields as of admin-console-parity.                                                                                   |
| ---                                              | ---                                                                                         | ---                                                                                                                                                                                                                                                                                                           |
| Change plate (step-up auth)                      | POST /api/admin/buses/{busId}/plate-change/authorize → PATCH /api/admin/buses/{busId}/plate | Step 1: caller's own login password (PlateChangeAuthorizeRequest) → short-lived unlockToken (PlateChangeAuthorizeResponse). Step 2: PlateUpdateRequest{newPlate, unlockToken}. Two-step by design - don't collapse into one call.                                                                             |
| ---                                              | ---                                                                                         | ---                                                                                                                                                                                                                                                                                                           |

### Checkpoints (route stops, managed from bus detail - no lat/lng, ordered labels only)

| Feature                                      | Endpoint                                 | Notes                                                                                                                                                                                                                                                                                                                                                       |
| -------------------------------------------- | ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| List checkpoints                             | GET /api/admin/buses/{busId}/checkpoints | ordered → CheckpointResponse\[\] (id, label, order)                                                                                                                                                                                                                                                                                                         |
| ---                                          | ---                                      | ---                                                                                                                                                                                                                                                                                                                                                         |
| Replace checkpoint list (add/reorder/remove) | PUT /api/admin/buses/{busId}/checkpoints | CheckpointRequest{checkpoints:\[{id?, label}\]} - send the **full desired list in drive order**. Items with id are updated in place; no id = new insert; omitted existing items are deleted **unless a student's checkpoint still references them** → 409 CHECKPOINT_IN_USE (response lists blocking students) - reassign those students' checkpoint first. |
| ---                                          | ---                                      | ---                                                                                                                                                                                                                                                                                                                                                         |

**Frontend note:** checkpoints must exist before importing/adding students (import validates each row's checkpoint label against this list → INVALID_CHECKPOINT / CHECKPOINT_NOT_FOUND otherwise).

| --------------------------------------------- | -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| ---                                           | ---                                    | ---                                                                                                                                                                                                                                                              |
| Driver detail                                 | GET /api/admin/drivers/{driverId}      | → DriverDetailResponse{id, name, phone, license, status, joined, busId, busDisplayId, routeName}                                                                                                                                                                 |

| ---                                           | ---                                    | ---                                                                                                                                                                                                                                                              |

## 3\. Student roster + parent linking (screens: **Admin Student** detail, **Admin Students** list, **Admin Parent(s)**)

Every student now carries a parent name, guardian phone, secondary phone (required), and a checkpoint (route stop label) instead of free-text stop. Adding/importing a student auto-creates-or-links its parent by phone (ParentStudent join) - no separate "create parent" call exists; parent creation is always a side effect of adding a student.

| Feature                                             | Endpoint                                              | Body / notes                                                                                                                                                                                                                                                                                                                                                                                                                                |
| --------------------------------------------------- | ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Add student to bus                                  | POST /api/admin/buses/{busId}/students                | StudentRequest{name\*, klass, parentName\*, phone\*, secondaryPhone\*, checkpoint}. phone = guardian/parent phone, used to find-or-create + link the parent. checkpoint must match a label from §2.                                                                                                                                                                                                                                         |
| ---                                                 | ---                                                   | ---                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Edit student                                        | PATCH /api/admin/students/{studentId}                 | StudentRequest, partial (no field required). Phone change re-links to the new parent and drops the stale link.                                                                                                                                                                                                                                                                                                                              |
| ---                                                 | ---                                                   | ---                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Remove student                                      | DELETE /api/admin/students/{studentId}                | Also removes the parent_student link row.                                                                                                                                                                                                                                                                                                                                                                                                   |
| ---                                                 | ---                                                   | ---                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **School-wide student list** (admin-console-parity) | GET /api/admin/students?klass=&busId=&status=&search= | Spans **all buses**, not just one - this is the standalone **Admin Students** screen. Search by student/parent name. status filters on attendanceStatus (below), **case-insensitive** - differs from the driver/trip endpoints.                                                                                                                                                                                                             |
| ---                                                 | ---                                                   | ---                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Student → response shape                            | -                                                     | StudentResponse{id, name, klass, parentName, phone, secondaryPhone, checkpointStopId, checkpoint, parentId, attendanceStatus}. attendanceStatus (ONBOARD/DROPPED/ABSENT/NOT*TODAY) is only populated by the list endpoint above - resolved from \_today's* trip on the student's bus; NOT_TODAY if no trip/attendance row exists yet. The bus-scoped roster endpoints (GET .../buses/{busId}, add/edit/delete above) leave this field null. |
| ---                                                 | ---                                                   | ---                                                                                                                                                                                                                                                                                                                                                                                                                                         |

### Parents (screens: **Admin Parents** list, **Admin Parent** detail) - standalone resource, admin-console-parity

A parent is still only ever _created_ as a side effect of adding a student (no direct "create parent" call). Once created, it's now a first-class resource:

| Feature               | Endpoint                                         | Body / notes                                                                                                                                                                                                         |
| --------------------- | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| List parents          | GET /api/admin/parents?search=                   | search by name/phone/child name → ParentSummaryResponse\[\]{id, name, phone, email, children\[\]{id, name, klass, busDisplayId}, accessStatus}. children is the parent's full linked-student list, batched (no N+1). |
| ---                   | ---                                              | ---                                                                                                                                                                                                                  |
| Parent detail         | GET /api/admin/parents/{parentId}                | → ParentDetailResponse{id, name, phone, email, accessStatus, children\[\]}                                                                                                                                           |

| ---                   | ---                                              | ---                                                                                                                                                                                                                  |

### Bulk onboarding via XLSX (per bus - replaces the old fleet-wide import)

Columns, exact order: name, klass, parent name, phone, secondary phone, checkpoint.

| Feature           | Endpoint                                                                                           | Notes                                                                                                                                                                                                                                      |
| ----------------- | -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Preview import    | POST /api/admin/buses/{busId}/students/import/preview (multipart file)                             | No writes. → StudentImportPreviewResponse{rows\[\], okCount, errorCount}, each row flags checkpoint/required-field errors.                                                                                                                 |
| ---               | ---                                                                                                | ---                                                                                                                                                                                                                                        |
| Commit import     | POST /api/admin/buses/{busId}/students/import/commit                                               | Body: { rows: \[...\] } - **send back the preview's rows array** (same shape), only ok rows are written. Two students sharing a phone in the same file collapse to one parent. → StudentImportCommitResponse{importedCount, skippedCount}. |
| ---               | ---                                                                                                | ---                                                                                                                                                                                                                                        |
| Export roster     | GET /api/admin/buses/{busId}/students/export                                                       | XLSX, 7 cols: Student ID (match key for bulk-edit) + the 6 import columns.                                                                                                                                                                 |
| ---               | ---                                                                                                | ---                                                                                                                                                                                                                                        |
| Preview bulk-edit | POST /api/admin/buses/{busId}/students/bulk-edit/preview (multipart file, re-upload of the export) | Matches rows by Student ID; unmatched/wrong-bus → row error. → StudentBulkEditPreviewResponse{rows\[\], okCount, errorCount}, each row includes changedFields\[\].                                                                         |
| ---               | ---                                                                                                | ---                                                                                                                                                                                                                                        |
| Commit bulk-edit  | POST /api/admin/buses/{busId}/students/bulk-edit/commit                                            | Body: { rows: \[...\] } from the preview. Phone change re-links parent. → StudentBulkEditCommitResponse{updatedCount, skippedCount}.                                                                                                       |
| ---               | ---                                                                                                | ---                                                                                                                                                                                                                                        |

**Frontend flow for both import and bulk-edit: preview → show diffs/errors to admin → confirm → commit with the same rows payload.** Never commit without a prior preview call - commit expects the preview's row objects, not raw file re-parse.

## 4\. Trips (screens: **Admin Trip**, **Admin Trips**)

Trip generation is automatic (lazy, one per bus per day) - no admin "create trip" action exists or is needed.

| Feature                                             | Endpoint                                   | Notes                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| --------------------------------------------------- | ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Get bus's active trip (dashboard/detail view)       | GET /api/trips/active/{busId}              | Role: ADMIN/SUPER_ADMIN/PARENT. Calling this **generates today's PENDING trip row if missing** (no scheduler) - safe to call on every dashboard load. Returns null data if today's trip hasn't started yet, otherwise TripResponse. Matches **only** status=ACTIVE - a cancelled or not-yet-restarted trip returns null here too, not just a never-started one.                                                                                                           |
| ---                                                 | ---                                        | ---                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Trip list/history** (admin-console-parity)        | GET /api/admin/trips?date=&status=&search= | Spans **all dates**, not just today - this is the standalone **Admin Trips** screen. Search by bus/route/driver name. status matches the TripStatus enum name exactly (case-sensitive): PENDING, ACTIVE, COMPLETED, CANCELLED. → TripSummaryResponse\[\]{id, date, shift, busDisplayId, routeName, driverName, start, end, studentCount, status, onTime}.                                                                                                                 |
| ---                                                 | ---                                        | ---                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Trip detail + roster** (admin-console-parity)     | GET /api/admin/trips/{tripId}              | → TripDetailResponse (summary fields above + roster: TripRosterEntryResponse\[\]{studentId, name, klass, checkpoint, attendanceStatus}). Roster = every student on the trip's bus; attendanceStatus defaults NOT_TODAY if no attendance row exists yet for this trip.                                                                                                                                                                                                     |
| ---                                                 | ---                                        | ---                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Cancel / uncancel a trip** (admin-console-parity) | POST /api/admin/trips/{tripId}/cancel      | Single toggle endpoint, both directions. PENDING/ACTIVE → CANCELLED; calling again on a CANCELLED trip restores PENDING. 400 TRIP_CANCEL_NOT_ALLOWED if the trip is COMPLETED. **Frontend must show a confirm dialog before calling this, in both directions - this is not re-confirmed server-side.**                                                                                                                                                                    |
| ---                                                 | ---                                        | ---                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| onTime field                                        | -                                          | true/false/null (null = trip hasn't started). Compares actual startTime (stamped by the driver's POST /api/trips/start, added by admin-console-parity - previously this field was dead code, always true) against a fixed 07:30 scheduled-departure placeholder. **Caveat: the comparison uses the API server's local time zone**, not the school's - if the deployment JVM isn't in the school's zone, onTime will be wrong. Flag to backend if schools span time zones. |
| ---                                                 | ---                                        | ---                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |

Driver-initiated POST /api/trips/start / /end exist but are DRIVER-only - not part of the admin surface, included for completeness only.

## Resolved gaps (admin-console-parity - closed since the previous version of this doc)

- **Driver/Parent/Trip/Student standalone list+detail endpoints** - all four now exist (§2a, §3, §4). Previously driver/parent were only reachable as fields on bus/student responses, and trips only had a single today's-active-trip lookup with no history, cancel state, or per-student attendance.
- **onTime was dead code** - always true in production because Trip.startTime only ever held the scheduled placeholder. POST /api/trips/start (driver-facing) now stamps the real start time, so onTime is meaningful (see the caveat in §4 about server time zone).
- **Trip cancellation** - didn't exist before; now a single toggle endpoint (§4), with CANCELLED added as a valid TripStatus.
- **School-wide student list** - students were only reachable per-bus; GET /api/admin/students (§3) now spans the whole school with class/bus/attendance-status filters.

## Known gaps to flag to frontend (don't silently work around)

- **Driver login is unsolved.** Drivers created via POST /api/admin/buses get a placeholder password - they cannot log in yet. Don't build a "driver login" flow against this.
- **No route/checkpoint coordinates.** Checkpoints are ordered labels only (no lat/lng) - map pins for stops aren't backed by data yet.
- **Fleet-wide import/export is gone.** /api/admin/fleet/\*\* no longer exists (removed this plan). All import/export is per-bus now (§3).
- **Precise pickup/dropoff coordinates** (pickupLat/Lng, dropoffLat/Lng on Student) exist as nullable DB columns only - no read/write endpoint yet. Driver-app calibration popup + parent/admin notification flow is designed but not built.
- **Status-filter vocabulary/case mismatch across endpoints.** Driver (§2a) and Trip (§4) status filters require the exact enum name, case-sensitive (ACTIVE, not active). Student (§3) status filter is case-insensitive. The v3 mockups use lowercase, different-per-screen vocabulary (active/leave/inactive for drivers, ongoing/completed/scheduled/cancelled for trips) - frontend must translate mockup labels to the actual enum names before sending, and match case where required.
- **onTime assumes the API server's time zone equals the school's.** See §4 - not validated against multi-timezone deployments.
